Does an object claim survive after the object is visually removed?

COCO-Ghost creates object-removed counterfactual images from COCO masks. GHOST-Guard accepts an object-presence claim only when the claim is supported by the original image, localized crop, and disappears after masking.

Safety wrapper behavior

Crop evidence alone is not enough

A crop-only gate confirms that the object can be recognized locally, but it does not test whether the claim disappears after removal. In the expanded runs, crop-gating still accepted most ghost claims. GHOST-Guard adds the masked counterfactual check and abstains when object belief persists.

Baseline finding

For qwen2.5vl:7b, crop-gating accepted 83.3% ghost claims. GHOST-Guard accepted 0.0% by converting those cases to UNSURE.

Method

Evidence survival under object removal

1

Select COCO instances

Sample target object categories with usable segmentation masks and moderate object area.

2

Create three views

Save original, object crop, and masked image with the target object region replaced by blurred local pixels.

3

Query local VLMs

Ask the same strict object-presence question through Ollama and preserve all raw responses.

4

Apply GHOST-Guard

Accept only original YES, crop YES, masked NO. Otherwise downgrade suspicious claims to UNSURE.

Qualitative examples

Object claims can persist after visual evidence removal

Some masked regions leave object-shaped blur artifacts. The benchmark treats this as an explicit artifact-audit issue and reports COCO-Ghost as a counterfactual stress test rather than proof that every masked-image YES is purely contextual.
