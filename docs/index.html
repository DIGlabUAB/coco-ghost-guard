COCO-Ghost: Auditing Whether VLM Object Claims Survive Object Removal

Dhiral Panjwani, M.S. · Anusha Kanagala, M.S. · Stephanie M. Aguilera, B.S. · Abhiraj Pudhota, M.S. · Leon Jololian, Ph.D. · Sandeep Bodduluri, M.S., Ph.D.

The basic question

Suppose a VLM says there is a cup in an image. Is it seeing the cup, or is it guessing from the surrounding scene? COCO-Ghost turns this into a counterfactual test: first verify that the model recognizes the object, then remove the object, and ask the same presence question again.

If the model still answers YES after the object is cleanly occluded, the claim has survived removal of the visual evidence. We call this a ghost. The point is not that every ghost is a deployment failure. The point is that the model's object claim is no longer locally supported by the image.

Experimental protocol

Three queries per object claim

1

Original image

Ask whether the target object is present in the full COCO image. This establishes that the model can make the object claim in context.

2

Object crop

Ask the same question on the crop. This checks whether the claim is supported by local object evidence rather than only by scene context.

3

Occluded image

Replace the target region with padded bounding-box solid local-mean fill, then ask again. A remaining YES is counted as a ghost.

What GHOST-Guard is useful for

GHOST-Guard is not a mechanistic explanation and it does not repair the VLM. It is a black-box abstention layer for object-presence claims. If the claim survives removal of the relevant visual evidence, a downstream application should not treat that claim as directly image-supported.

In practice, this can trigger escalation: ask a detector, query a stronger model, request another view, or send the case to a human reviewer. The contribution is a simple reliability check that can be run without access to model weights or activations.

Guard behavior

0.0% of detected ghost claims were accepted by GHOST-Guard in the reported Qwen2.5-VL 7B analysis.

51.5% abstention rate reflects the cost of rejecting unsupported claims.

Scope and limitations

COCO-Ghost is a counterfactual stress test for object-presence claims. It does not claim that every masked-image YES is a human-level hallucination, and it does not evaluate all forms of VLM reasoning. Its strength is narrowness: it asks one question cleanly, with artifact labels and results reported separately for clean-removal cases.
